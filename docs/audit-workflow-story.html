US User Story Analisada

Quando qualquer usuario carrega o dashboard, um modal obrigatorio aparece oferecendo: (1) iniciar uma nova tarefa baseada em templates de fluxo publicados, ou (2) completar uma tarefa pendente atribuida a ele. O sistema de workflow e o nucleo da plataforma.

04 Diagrama de Sequencia — Jornada do Usuario 
sequenceDiagram
    autonumber
    actor U as Usuario
    participant D as Dashboard
    participant M as TaskActionModal
    participant F as /fluxos
    participant T as /minhas-tarefas
    participant API as Backend API
    participant DB as PostgreSQL

    U->>D: Acessa dashboard
    D->>API: GET /api/workflow/tasks/mine/
    API->>DB: Query TaskInstance (pending)
    DB-->>API: tasks[]
    API-->>D: pending tasks + templates
    D->>M: Exibe modal obrigatorio

    alt Opcao A: Nova Tarefa
        U->>M: Clica "Iniciar nova tarefa"
        M->>F: Navega para /fluxos
        U->>F: Seleciona template
        F->>API: POST /api/workflow/start/
        API->>DB: Cria FlowInstance
        API->>DB: Cria TaskInstance
        DB-->>API: instance criada
        API-->>F: Redireciona para execucao
    else Opcao B: Tarefa Pendente
        U->>M: Clica "Finalizar tarefa"
        M->>T: Navega para /minhas-tarefas
        U->>T: Seleciona tarefa pendente
        T->>API: POST /api/workflow/tasks/{id}/complete/
        API->>DB: Atualiza TaskInstance
        API->>DB: Avanca FlowInstance
        DB-->>API: fluxo avancado
        API-->>T: Confirmacao
    end
05 Fases da Jornada
1
Login
Usuario autentica via JWT. Role e organ sao carregados.
2
Dashboard
Pagina principal carrega. API retorna tarefas e templates.
3
TaskModal
Modal obrigatorio aparece com duas opcoes de acao.
4
Iniciar Fluxo
Seleciona template publicado e inicia FlowInstance.
5
Executar
TaskInstances sao atribuidas. Usuario preenche formularios.
6
Completar
Tarefa concluida. Motor avanca fluxo para proximo no.
